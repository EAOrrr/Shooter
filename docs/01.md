# 01 · Bounding Box & Collision Primitives · `[Domain Primitives]`

**Start:** `git switch main && git pull && git switch -c feat/01-collision-primitives`

**Story.** As a developer, I want a robust `GameObject` base class with bounding-box collision detection and delta-time updates so that all game entities can update state frame-independently.

**Acceptance criteria**

* AC1 `GameObject` holds coordinates $(x, y)$, dimensions $(w, h)$, and boolean state `active`.
* AC2 Abstract method `update(double delta)` defined on `GameObject`, where `delta` represents elapsed time in seconds (e.g., $0.016$s for ~60 FPS).
* AC3 `intersects(GameObject other)` performs Axis-Aligned Bounding Box (AABB) intersection check.

**Test Coverage Cases** (`CollisionTest.java`)

* **Case 1 (AABB Overlap):** Box A $(0, 0, 20, 20)$ and Box B $(10, 10, 20, 20)$ $\rightarrow$ `intersects()` returns `true`.
* **Case 2 (AABB Separation):** Box A $(0, 0, 20, 20)$ and Box B $(50, 50, 20, 20)$ $\rightarrow$ `intersects()` returns `false`.
* **Case 3 (Zero Delta Safety):** Calling `update(0.0)` on a `GameObject` $\rightarrow$ maintains identical position without throwing exceptions.

**Tasks**

1. Setup Maven project structure (`src/main/java`, `src/test/java`) with JUnit 5 dependency.
2. Implement `GameObject` abstract base class with `public abstract void update(double delta)`.
3. Write and run `CollisionTest`.

**Done.** `mvn test` green, core math and delta-based object architecture verified.
