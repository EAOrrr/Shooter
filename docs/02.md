
# 02 · Player HP Lifecycle & Frame-Independent Movement · `[Domain Entity]`

**Start:** `git switch main && git pull && git switch -c feat/02-player-entity`

**Story.** As a player, I want a `Player` airplane whose movement scales smoothly with `delta` time and whose HP lifecycle is bounded.

**Acceptance criteria**

* AC1 `AirPlane` extends `GameObject`, adding `hp` and `maxHp`.
* AC2 Player position updates scale with `delta`: $x \leftarrow x + (speedX \times delta)$.
* AC3 `takeDamage(damage)` reduces `hp`. If `hp <= 0`, set `active = false`.
* AC4 Player movement is clamped within window bounds ($x \in [0, \text{WIDTH} - w]$, $y \in [0, \text{HEIGHT} - h]$).

**Test Coverage Cases** (`PlayerTest.java`)

* **Case 1 (Delta-Based Movement Velocity):** Player at $x = 100$ with $speedX = 200\text{ px/s}$, updated with `delta = 0.5`s $\rightarrow$ new $x = 200$.
* **Case 2 (Damage Calculation):** $100$ HP player receives $30$ damage $\rightarrow$ `hp = 70`, `active = true`.
* **Case 3 (Fatal Damage State):** $100$ HP player receives $100$ damage $\rightarrow$ `hp = 0`, `active = false`.
* **Case 4 (Boundary Clamp with Large Delta):** Moving player left at $x = 10$ with $speedX = -500\text{ px/s}$ and `delta = 1.0`s $\rightarrow$ $x$ clamped at $0$.

**Tasks**

1. Create `AirPlane` abstract class extending `GameObject`.
2. Create `Player` class implementing `update(double delta)` with boundary clamping.
3. Write unit tests for velocity/delta math.

**Done.** `mvn test` green, Player velocity scales precisely with delta time.
