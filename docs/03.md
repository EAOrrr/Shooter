
# 03 · Delta Game Loop & Render Container · `[UI Foundation]`

**Start:** `git switch main && git pull && git switch -c feat/03-game-loop-ui`

**Story.** As a player, I want a Swing window running a high-precision game loop that calculates real `delta` time per frame and passes it to entity updates.

**Acceptance criteria**

* AC1 `GamePanel` initializes a $400 \times 600$ pixel canvas.
* AC2 Game loop calculates $delta = \frac{currentTime - lastTime}{1,000,000,000.0}$ in seconds per tick.
* AC3 `KeyInput` captures keyboard state and updates Player velocity vectors.

**Test Coverage Cases** (`InputHandlingTest.java`)

* **Case 1 (Key Down Sets Velocity Vector):** Pressing `A` key $\rightarrow$ sets `velocityX = -SPEED_PX_PER_SEC` (e.g. $-300.0$).
* **Case 2 (Key Up Clears Velocity Vector):** Releasing `A` key $\rightarrow$ resets `velocityX = 0.0`.

**Tasks**

1. Implement `KeyInput` listening for `WASD`/Arrow keys.
2. Implement `GamePanel` game loop passing calculated `delta` into `player.update(delta)`.
3. Implement `Main` entry point with `JFrame`.

**Done.** App runs, player moves smoothly with velocity calculated against actual elapsed frame delta.
