
# 04 · Multi-Stream SimpleShot & Delta Cooldown · `[Weapon System I]`

**Start:** `git switch main && git pull && git switch -c feat/04-simple-shot-weapon`

**Story.** As a player, I want my plane to automatically fire parallel straight bullets driven by a delta-accumulating cooldown timer.

**Acceptance criteria**

* AC1 `Weapon` interface defines `shoot(double x, double y, boolean isPlayer)` returning `List<Bullet>`.
* AC2 `SimpleShotWeapon` initializes with `count` (parallel streams) and `maxWidth` (horizontal span).
* AC3 Cooldown Accumulator: `cooldownTimer += delta`. When `cooldownTimer >= cooldownInterval`, firing is triggered and timer resets.
* AC4 Bullets traveling past screen boundaries ($y < 0$ or $y > \text{HEIGHT}$) automatically deactivate (`active = false`).

**Test Coverage Cases** (`WeaponTest.java`, `BulletTest.java`)

* **Case 1 (Multi-Stream Parallel X Positions):** `SimpleShotWeapon` (`count = 3`, `maxWidth = 20`) fired at $x = 100$ $\rightarrow$ 3 bullets returned at $X = [90, 100, 110]$.
* **Case 2 (Delta Cooldown Trigger):** Weapon with `cooldownInterval = 0.2`s:
* `update(0.1)` $\rightarrow$ no shooting (`cooldownTimer = 0.1`).
* Next `update(0.15)` $\rightarrow$ triggers shoot (`cooldownTimer` resets).


* **Case 3 (Bullet Velocity via Delta):** Bullet at $y = 500$ with $speedY = -400\text{ px/s}$, `update(0.5)` $\rightarrow$ new $y = 300$.
* **Case 4 (Off-screen Deactivation):** Bullet at $y = -5$ updated $\rightarrow$ `active` becomes `false`.

**Tasks**

1. Create `Weapon` interface and `SimpleShotWeapon` class with delta timer accumulator.
2. Create `Bullet` entity class updating position via `y += speedY * delta`.
3. Integrate auto-shooting in `GamePanel` and clear inactive bullets.

**Done.** `mvn test` green, auto-shooting fires parallel vertical bullets at exact time intervals regardless of frame rate.
