

# 05 · Delta Enemy Spawning & Collision Resolution · `[Game Loop Core]`

**Start:** `git switch main && git pull && git switch -c feat/05-enemy-collision`

**Story.** As a player, I want enemy planes to spawn at time intervals and descend based on delta time so that I can shoot them down.

**Acceptance criteria**

* AC1 `SimpleEnemy` spawns at $y = -30$ and descends at $speedY \times delta$.
* AC2 Enemy Spawner uses delta accumulator (`spawnTimer += delta`) to spawn enemies every $N$ seconds.
* AC3 Collision between Player Bullet and `SimpleEnemy` deactivates both and increases `score` by $10$.

**Test Coverage Cases** (`EnemyCollisionTest.java`)

* **Case 1 (Enemy Delta Movement):** Enemy at $y = 0$ with $speedY = 100\text{ px/s}$, `update(0.5)` $\rightarrow$ $y = 50$.
* **Case 2 (Bullet Kills Enemy):** Active bullet overlaps active enemy $\rightarrow$ both marked `active = false`, `score += 10`.
* **Case 3 (Enemy Escapes Bottom):** Enemy updated past $y = 610$ $\rightarrow$ marked `active = false`.

**Tasks**

1. Implement `Enemy` abstract class and `SimpleEnemy` implementation.
2. Add delta-based enemy spawner in `GamePanel`.
3. Implement collision resolution in `GamePanel.checkCollisions()`.

**Done.** `mvn test` green, enemies descend predictably and collisions award score properly.
