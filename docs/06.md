
# 06 · Spread Weapon Trigonometry & Strategy Swap · `[Weapon System II]`

**Start:** `git switch main && git pull && git switch -c feat/06-spread-weapon`

**Story.** As a player, I want a fan-shaped multi-angle weapon whose bullet trajectories and cooldowns update via delta time.

**Acceptance criteria**

* AC1 `SpreadWeapon` accepts `count` and `maxAngle`.
* AC2 Calculates bullet velocities using trigonometry:
* $speedX = speed \times \sin(\theta)$
* $speedY = -speed \times \cos(\theta)$


* AC3 `player.upgradeWeapon()` swaps active `Weapon` strategy seamlessly.

**Test Coverage Cases** (`SpreadWeaponTest.java`)

* **Case 1 (Symmetric Velocity Vectors):** 3-shot 30° spread with $speed = 500$:
* Left bullet: $speedX = -500 \times \sin(15^\circ) \approx -129.4$
* Center bullet: $speedX = 0$, $speedY = -500$
* Right bullet: $speedX = +500 \times \sin(15^\circ) \approx +129.4$


* **Case 2 (Delta Movement along Angle):** Bullet with $speedX = 100$, $speedY = -200$, `update(0.5)` $\rightarrow$ $\Delta x = +50$, $\Delta y = -100$.
* **Case 3 (Weapon Strategy Mutation):** Calling `upgradeWeapon()` replaces `SimpleShotWeapon` with `SpreadWeapon`.

**Tasks**

1. Implement `SpreadWeapon` with angle distribution logic.
2. Update `Bullet.update(double delta)` to apply both $speedX \times delta$ and $speedY \times delta$.
3. Add `upgradeWeapon()` to `Player`.

**Done.** `mvn test` green, fan-shaped bullets travel smoothly along angled trajectory vectors.
