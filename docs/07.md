
# 07 · Boss Multi-Phase Delta Mechanics & Debug Tools · `[Boss & Tools]`

**Start:** `git switch main && git pull && git switch -c feat/07-boss-and-debug`

**Story.** As a developer, I want a multi-stage Boss with delta-driven movement patterns and developer debug hotkeys (`F1-F4`).

**Acceptance criteria**

* AC1 `BossEnemy` moves horizontally using a sine/cosine wave calculated against accumulated delta time ($\text{timeAccumulator} += delta$).
* AC2 Phase Transition: When Boss HP drops below $50\%$ ($<250$ HP), its internal weapon mutates to `SpreadWeapon`.
* AC3 Debug hotkeys: `F1` (God Mode), `F2` (Instant Upgrade), `F3` (Show Hitboxes), `F4` (Spawn Boss).

**Test Coverage Cases** (`BossTest.java`, `DebugModeTest.java`)

* **Case 1 (Boss Phase 1 Weapon):** Boss at $100\%$ HP $\rightarrow$ `weapon` is `SimpleShotWeapon`.
* **Case 2 (Boss Phase 2 Mutation):** Boss HP drops to $240$ after `takeDamage()` $\rightarrow$ `weapon` mutates to `SpreadWeapon`.
* **Case 3 (Boss Delta Oscillation):** `update(0.5)` advances Boss position along smooth horizontal trajectory.
* **Case 4 (God Mode Immunity):** `player.setInvincible(true)` $\rightarrow$ `takeDamage(50)` leaves HP unchanged.

**Tasks**

1. Implement `BossEnemy` with delta-based trajectory and phase transition check in `update(double delta)`.
2. Add `showHitbox` and `invincible` flags in `GamePanel` / `Player`.
3. Wire `F1-F4` in `KeyInput`.

**Done.** `mvn test` green, Boss oscillates smoothly using delta time and shifts patterns at $50\%$ HP.
