
# 08 · Resource Management & Fat-JAR Build · `[Assets & Packaging]`

**Start:** `git switch main && git pull && git switch -c feat/08-resources-and-packaging`

**Story.** As a user, I want the game packaged into a single executable Fat-JAR with Classpath-loaded assets.

**Acceptance criteria**

* AC1 Assets loaded via `ClassLoader.getResourceAsStream()`.
* AC2 Fallback: Missing images default gracefully to vector shape rendering.
* AC3 `mvn clean package` runs all JUnit 5 tests and outputs executable JAR in `target/`.

**Test Coverage Cases** (`ResourceLoaderTest.java`)

* **Case 1 (Classpath Stream Non-Null):** `ImageManager.class.getResourceAsStream("/images/player.png")` returns non-null stream.
* **Case 2 (Missing Asset Graceful Fallback):** Requesting non-existent image returns `null` without throwing unhandled exception.

**Tasks**

1. Create `ImageManager` using `getResourceAsStream`.
2. Update `draw(Graphics g)` methods to render sprite images when present, falling back to colored shapes.
3. Configure `maven-shade-plugin` in `pom.xml`.
4. Run `mvn clean package` and verify execution via `java -jar target/plane-game-1.0.0.jar`.

**Done.** `mvn clean package` yields `BUILD SUCCESS`, JUnit test suite runs 100% green, and double-clicking the shaded JAR launches the game cleanly!
